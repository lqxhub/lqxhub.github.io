linux 多线程或多进程 epoll处理 accept 导致惊群

发表于 2021-08-22 更新于 2024-03-10 分类于 C++
本文字数： 2.6k 阅读时长 ≈ 9 分钟

简单说，惊群是因为多进程（多线程）在同时阻塞等待同一个事件的时候（休眠状态），当事件发生时，就会唤醒所有等待的（休眠的）进程（线程）。但是事件只能被一个进程或线程处理，而其他进程（线程）获取失败，只能重新进入休眠状态，这种现象和性能浪费就叫做惊群
